They are two types of accounts
1. User Account:
    This is an account used by humans to access to the cluster for administrative task or development task.

2. Service Account
    This is an account used by bot to access the kube-api server, example is a monitoring service like prometheus
    needs to query the kube-api server for the health and metric of the cluster.


 kubectl create sa sa_name

 kubectl get sa


 When a service account is created, a token is created as a secrets, this secret object is then linked to the
 service account and can used in a pod as a mount volume or use this token to access the kube-api server.

 When you mount it as a pod, it means the pod is inside the kubernetes cluster, so it means that the pod would
 use it.
